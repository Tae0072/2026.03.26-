# 1. CSS Flexbox와 Grid 레이아웃의 차이점을 설명하고, 각각이 적합한 UI 패턴과 구현 코드를 작성하시오.

## 개념 비교

| 구분          | Flexbox                              | Grid                               |
| ------------- | ------------------------------------ | ---------------------------------- |
| 레이아웃 방향 | **1차원** (행 또는 열 중 하나)       | **2차원** (행 + 열 동시)           |
| 주 사용 목적  | 아이템 간 정렬·배분                  | 전체 페이지/영역 레이아웃 설계     |
| 배치 기준     | 콘텐츠 크기에 따라 유동적            | 명시적 격자(track)에 배치          |
| 브라우저 처리 | 자식 요소 중심으로 계산              | 부모 컨테이너 격자 기준으로 계산   |
| 적합한 패턴   | 네비게이션 바, 버튼 그룹, 카드 한 줄 | 대시보드, 갤러리, 전체 페이지 구조 |

## Flexbox — 1차원 정렬

```html
<!-- 네비게이션 바 예시 -->
<nav class="navbar">
  <div class="logo">MyApp</div>
  <ul class="nav-links">
    <li><a href="#">홈</a></li>
    <li><a href="#">소개</a></li>
    <li><a href="#">연락처</a></li>
  </ul>
</nav>
```

```css
.navbar {
  display: flex;
  justify-content: space-between; /* 로고와 메뉴를 양 끝으로 */
  align-items: center; /* 수직 가운데 정렬 */
  padding: 0 24px;
  height: 60px;
  background-color: #333;
}

.nav-links {
  display: flex;
  gap: 24px;
  list-style: none;
}

.nav-links a {
  color: #fff;
  text-decoration: none;
}
```

## Grid — 2차원 레이아웃

```html
<!-- 대시보드 레이아웃 예시 -->
<div class="dashboard">
  <header class="header">헤더</header>
  <aside class="sidebar">사이드바</aside>
  <main class="content">메인 콘텐츠</main>
  <footer class="footer">푸터</footer>
</div>
```

```css
.dashboard {
  display: grid;
  grid-template-columns: 220px 1fr; /* 사이드바 고정, 나머지 유동 */
  grid-template-rows: 60px 1fr 50px; /* 헤더, 본문, 푸터 */
  grid-template-areas:
    "header  header"
    "sidebar content"
    "footer  footer";
  height: 100vh;
  gap: 8px;
}

.header {
  grid-area: header;
  background: #4a90e2;
}
.sidebar {
  grid-area: sidebar;
  background: #f0f0f0;
}
.content {
  grid-area: content;
  background: #fff;
}
.footer {
  grid-area: footer;
  background: #333;
}
```

# 2. JavaScript의 비동기 처리 방식(Callback, Promise, async/await)을 비교 설명하고, fetch API를 활용한 데이터 통신 패턴을 서술하시오.

## ① Callback

가장 오래된 비동기 패턴. 함수의 인자로 다음 작업을 전달합니다.

```js
function fetchData(url, onSuccess, onError) {
  const xhr = new XMLHttpRequest();
  xhr.open("GET", url);
  xhr.onload = () => {
    if (xhr.status === 200) onSuccess(JSON.parse(xhr.responseText));
    else onError(new Error("요청 실패"));
  };
  xhr.send();
}

// 문제: 중첩이 깊어지면 "Callback Hell" 발생
fetchData(
  "/api/user",
  (user) => {
    fetchData(
      `/api/posts/${user.id}`,
      (posts) => {
        fetchData(
          `/api/comments/${posts[0].id}`,
          (comments) => {
            console.log(comments); // 콜백 지옥
          },
          console.error,
        );
      },
      console.error,
    );
  },
  console.error,
);
```

---

## ② Promise

ES6에서 도입. 비동기 작업의 성공/실패를 `.then()` / `.catch()`로 체이닝합니다.

```js
function fetchData(url) {
  return new Promise((resolve, reject) => {
    fetch(url)
      .then((res) => {
        if (!res.ok) reject(new Error("요청 실패"));
        return res.json();
      })
      .then(resolve)
      .catch(reject);
  });
}

// 체이닝으로 콜백 지옥 해결
fetchData("/api/user")
  .then((user) => fetchData(`/api/posts/${user.id}`))
  .then((posts) => fetchData(`/api/comments/${posts[0].id}`))
  .then((comments) => console.log(comments))
  .catch((err) => console.error("에러:", err));
```

## ③ async/await

ES2017에서 도입. Promise를 동기 코드처럼 읽을 수 있게 해줍니다.

```js
async function loadComments() {
  try {
    const user = await fetch("/api/user").then((r) => r.json());
    const posts = await fetch(`/api/posts/${user.id}`).then((r) => r.json());
    const comments = await fetch(`/api/comments/${posts[0].id}`).then((r) =>
      r.json(),
    );
    console.log(comments);
  } catch (err) {
    console.error("에러:", err);
  }
}

loadComments();
```

## fetch API를 활용한 데이터 통신 패턴

```js
// GET 요청
async function getUsers() {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  if (!res.ok) throw new Error(`HTTP Error: ${res.status}`);
  return res.json();
}

// POST 요청
async function createPost(data) {
  const res = await fetch("https://jsonplaceholder.typicode.com/posts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(data),
  });
  if (!res.ok) throw new Error(`HTTP Error: ${res.status}`);
  return res.json();
}

// 병렬 요청 (Promise.all)
async function loadDashboard() {
  const [users, posts] = await Promise.all([
    fetch("/api/users").then((r) => r.json()),
    fetch("/api/posts").then((r) => r.json()),
  ]);
  console.log(users, posts);
}
```

## 비교 요약

| 방식        | 가독성        | 에러 처리   | 사용 권장      |
| ----------- | ------------- | ----------- | -------------- |
| Callback    | 낮음 (중첩)   | 복잡        | 레거시 코드    |
| Promise     | 중간 (체이닝) | `.catch()`  | 라이브러리 API |
| async/await | 높음 (동기식) | `try/catch` | 현대 개발 권장 |

# 3. 디바운싱(Debouncing)과 쓰로틀링(Throttling)의 개념과 차이점을 설명하고, 검색 입력창에 디바운싱을 적용하는 코드를 작성하시오.

## 개념

**디바운싱(Debouncing)**
연속으로 발생하는 이벤트 중 **마지막 이벤트만** 실행합니다.
지정한 대기 시간(delay) 내에 새 이벤트가 오면 타이머를 초기화합니다.

**쓰로틀링(Throttling)**
연속 이벤트를 **일정 간격으로 제한**하여 실행합니다.
지정한 주기마다 최대 한 번만 함수를 호출합니다.

---

## 차이점 비교

| 구분        | 디바운싱                        | 쓰로틀링                       |
| ----------- | ------------------------------- | ------------------------------ |
| 실행 시점   | 이벤트가 멈춘 후                | 일정 주기마다                  |
| 실행 횟수   | 연속 이벤트 중 1회              | 주기당 1회                     |
| 적합한 상황 | 검색 자동완성, 입력 유효성 검사 | 스크롤 이벤트, 윈도우 리사이즈 |

---

## 검색 입력창에 디바운싱 적용

```html
<div class="search-wrap">
  <input type="text" id="searchInput" placeholder="검색어를 입력하세요" />
  <p id="result">결과가 여기에 표시됩니다.</p>
</div>
```

```js
// 디바운싱 유틸리티 함수
function debounce(fn, delay) {
  let timer = null;

  return function (...args) {
    // 새 이벤트가 오면 기존 타이머 초기화
    clearTimeout(timer);

    // delay 후 실행
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 실제 검색 API 호출 함수
async function searchAPI(query) {
  const result = document.getElementById("result");
  if (!query) {
    result.textContent = "결과가 여기에 표시됩니다.";
    return;
  }

  result.textContent = `"${query}" 검색 중...`;

  try {
    // 예시: JSONPlaceholder API
    const res = await fetch(
      `https://jsonplaceholder.typicode.com/posts?title_like=${query}`,
    );
    const data = await res.json();
    result.textContent = `"${query}" 결과: ${data.length}건`;
  } catch (err) {
    result.textContent = "검색 중 오류가 발생했습니다.";
  }
}

// 300ms 디바운싱 적용
const debouncedSearch = debounce(searchAPI, 300);

document.getElementById("searchInput").addEventListener("input", (e) => {
  debouncedSearch(e.target.value.trim());
});
```
